# GAIIC2024 RGB-TIR 检测优化实验报告（optimized 版本）

## 摘要
本次优化围绕 RGB-TIR 检测中的三类核心问题展开：跨场景泛化不足、跨模态成像差异导致的鲁棒性下降、以及训练链路稳定性不足。整体策略采用低侵入路线：优先强化数据侧鲁棒性建模，修复关键可训练性阻塞，再通过训练流程稳健化降低非模型因素干扰。该方案在不进行高风险结构重构的前提下，为后续结构升级建立了可复现的对比基线。

## 1. 研究背景与问题定义
从任务特性看，RGB-TIR 检测面临以下典型难点：
1. 场景复杂且光照变化剧烈，模型在跨域场景上容易出现性能波动。
2. RGB 与 TIR 在成像机理上存在天然差异，且伴随轻度配准偏差、模糊与噪声。
3. 训练效果会受到工程链路稳定性的显著影响，造成实验收益难以准确归因。

结合参考论文 2407.03872v1 的结论，数据增强在该类任务中是高杠杆优化项，因此本次优先从数据和训练链路入手。

## 2. 优化目标
本次优化目标分为三层：
1. 数据层目标：提升对模态差异、噪声与弱配准的耐受性。
2. 训练层目标：消除会直接阻断训练的配置级风险点。
3. 工程层目标：提高训练流程可重复执行能力，减少环境和路径因素造成的伪波动。

## 3. 优化方法与具体改造

### 3.1 模态专属数据增强
针对“同一套增强策略无法同时匹配 RGB 与 TIR 退化特征”的问题，采用模态专属增强方案：
1. RGB 分支引入亮度/对比度扰动与噪声扰动，重点模拟光照与成像动态变化。
2. TIR 分支引入模糊、边缘增强与噪声扰动，重点模拟红外通道中细节弱化与成像不稳定。
3. 保留几何类扰动（平移、翻转、旋转、裁剪）以持续覆盖配准误差带来的空间偏移风险。

该策略的核心思想是“按模态建模退化分布”，而不是单纯堆叠统一增强算子。

### 3.2 两阶段增强调度
为兼顾鲁棒性学习与后期收敛稳定性，采用两阶段训练策略：
1. 阶段一：高强度模态专属扰动，优先扩大有效训练分布并提升抗干扰能力。
2. 阶段二：降低增强强度，减少后期优化噪声，提升收敛质量。

这一路线可避免“全程强增强导致后期收敛受扰”的常见问题。

### 3.3 关键可训练性修复
针对训练图构建阶段的阻塞风险，补全了 denoising query 生成器的关键配置项，确保模型在该配置分支下可被正确实例化并进入训练。

该改动属于“必要条件修复”：不直接提高上限，但对训练可执行性至关重要。

### 3.4 训练链路稳健化
为降低非模型因素带来的实验偏差，对训练流程做了稳健化处理：
1. fail-fast 机制，防止错误被静默放大。
2. 分布式入口统一绑定当前环境解释器，避免多环境错配。
3. 数据软链接与权重下载改为可重入，支持重复执行与中断恢复。
4. 数据路径变量统一，避免因变量引用不一致导致的数据分片失败。

这些改动提升了实验可复现性，使模型侧优化收益更容易被准确观测。

## 4. 与论文思路的对应关系
本次优化与 2407.03872v1 的关系可以概括为：
1. 直接继承了“按模态差异设计增强策略”的思路，并转化为工程可复用模块。
2. 贯彻了“先做高确定性收益项”的策略，即优先做数据层鲁棒性增强而非立即进行结构重构。
3. 当前版本未引入高风险结构升级，目的是先稳定训练基线，再开展结构级增量验证。

## 5. 预期效果与风险

### 5.1 预期效果
1. 提升跨场景与跨时段鲁棒性。
2. 降低模态成像差异引发的误检和漏检。
3. 提升训练过程稳定性与复现实验效率。

### 5.2 主要风险
1. 增强强度若设置过高，可能引入后期欠拟合。
2. 不同数据版本下，最优增强概率和强度存在迁移偏差。
3. 工程稳健化提升了“可跑性”，但模型上限仍需后续结构实验验证。

## 6. 结论
本次 optimized 版本的核心贡献是建立了“问题导向、低侵入、可复现”的优化闭环：
1. 用模态专属增强解决数据侧失配问题。
2. 用关键配置修复保障训练可执行性。
3. 用流程稳健化减少非模型因素噪声。

该版本为下一阶段结构升级实验提供了更干净、可信的对照基础。
